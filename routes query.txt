Покупки/Продажи
GET запрос - api/sales - Получить все сделки (только для админа),
GET запрос - api/sale/:adsId - Сделка между покупателем и продавцом, сохраняет их данные и название товара и цену(принимает айди покупателя),
GET запрос - api/sale/buyer - Получить все сделки от лица покупателя вещи которые купил (только для авторизированых пользователей),
GET запрос - api/sale/seller - Получить все сделки от лица продавца вещи которые продал (только для авторизированых пользователей),

Объявления
GET запрос - api/ads/ - Получить все объявления, возможна пагинация.
GET запрос - api/ads/allForUser - Получить все объявления пользователя(только для авторизированых пользователей),
GET запрос - api/ads/:adsId - Получить объявление по айди(только для авторизированых пользователей),
POST запрос - api/ads/ - Создать объявление {title, price, description}(только для авторизированых пользователей),
PATCH запрос - api/ads/:adsId/fotoAds - добавление фотографии к объявлению принимает айди обявление вложить в поле под именем fotoAds файл,
PUT запрос - api/ads/:adsId - обновление объявления по полям {title, price, description} нужно передать айди объявления,
DELETE запрос - api/ads/:adsId - удаление объявления, нужно передать айди объявления,
PATCH запрос - api/ads/reset-views/:adsId - сброс количество просмотров, нужно передать айди объявления,
PACTH запрос - api/ads/active/:adsId - активировать объявление, нужно передать айди объявления,

Пользователь
GET запрос - api/users/ - получить всех зарегестрированных юзеров (только для админа),
DELETE запрос - api/users/:userId - удалить юзера с его объявлениями (только для админа), принимает айди юзера,
POST запрос - api/users/register - Зарегестрировать пользователя {name, phone, email, password},
GET запрос - api/users/verify/:verificationToken - Верификация, приходит письмо на почту с ссылкой,
POST запрос - api/users/verify - отправить письмо повторно нужно отправить почту {email},
POST запрос - api/users/login - авторизироватся нужно отправить {email, password} в ответ получите токен,
GET запрос - api/users/current - если есть токен, при этом запросе получите данные владельца токена {name, email}
POST запрос - api/users/logout - если есть токен, нужно отправить {token: null}
PATCH запрос - api/users/avatars - добавление автара, нужно вложить в поле под именем avatar файл,
PATCH запрос - api/users/admin - получения права администратора, нужно отправить { adminAccess: true, passwordAdmin: password }
